# 跨文件类型推断说明

## 问题回答

**问题：解析调用的参数类型时用到索引中其他类的 return_type 了吗？**

**回答：是的，已经实现了，但使用方式与预期可能不同。**

## 当前实现方式

### 1. 返回类型已存储在索引中 ✅

所有方法的返回类型都已经存储在索引中，可以通过 `find_method_return_type` API 查询：

```rust
// 查询任何方法的返回类型（包括其他文件中的方法）
let return_type = index.find_method_return_type("com.example.UserRepository::findUser(String)");
// 返回: Some("User")
```

### 2. 文件内类型推断（解析时） ✅

在解析单个文件时，使用文件内的方法返回类型映射进行类型推断：

```java
// 同一文件内
public class UserRepository {
    public User findUser(String id) { return null; }
}

public class TestService {
    private UserRepository repo;
    
    public void test() {
        // findUser() 返回 User，参数类型被正确推断为 User
        processor.process(repo.findUser("123"));
        // 方法调用目标：Processor::process(User)
    }
}
```

### 3. 跨文件类型查询（使用时） ✅

在影响分析时，可以使用索引查询跨文件的返回类型：

```rust
// 在影响分析时
let method = index.find_method("com.example.Service::process").unwrap();
for call in &method.calls {
    // 查询被调用方法的返回类型（可能在其他文件中）
    if let Some(return_type) = index.find_method_return_type(&call.target) {
        println!("调用 {} 返回 {}", call.target, return_type);
    }
}
```

## 为什么不在解析时使用索引？

### 原因 1：避免循环依赖

- 解析文件 A 需要文件 B 的返回类型
- 但文件 B 可能还未被解析
- 或者文件 B 也需要文件 A 的返回类型

### 原因 2：支持并行解析

- 当前使用 rayon 并行解析所有文件以提高性能
- 多个文件同时被解析时，索引处于不完整状态
- 解析过程中查询索引会得到不完整的结果

### 原因 3：性能考虑

如果要在解析时使用索引，需要：
1. 第一遍：解析所有文件，只提取方法签名和返回类型
2. 第二遍：使用完整索引重新解析所有文件

这会使解析时间翻倍。

## 实际使用示例

### 场景：追踪跨文件调用链

```rust
// 文件 A: UserRepository.java
public class UserRepository {
    public User findUser(String id) { return null; }
}

// 文件 B: UserService.java
public class UserService {
    private UserRepository repo;
    
    public void process(String userId) {
        processor.handle(repo.findUser(userId));
    }
}

// 使用索引查询
let repo_method = "com.example.UserRepository::findUser(String)";
let return_type = index.find_method_return_type(repo_method);
// 返回: Some("User")

// 在影响分析中使用
let service_method = index.find_method("com.example.UserService::process").unwrap();
for call in &service_method.calls {
    if let Some(return_type) = index.find_method_return_type(&call.target) {
        println!("调用 {} 返回 {}", call.target, return_type);
    }
}
```

## 总结

1. **已实现**：返回类型存储在索引中，可以查询跨文件的返回类型
2. **使用方式**：在影响分析时使用 `find_method_return_type` API 查询
3. **设计权衡**：解析时使用文件内类型推断，使用时查询索引获取跨文件类型
4. **性能优势**：避免两遍解析，保持高性能并行解析

这是一个合理的设计决策，在性能和功能之间取得了平衡。
