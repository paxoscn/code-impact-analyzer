# 集合工厂方法识别功能

## 功能说明

实现了对常见Java集合工厂方法的返回类型识别，使得方法调用的参数类型推断更加精确。

## 示例

### 之前
```java
processList(Lists.newArrayList());  // 识别为: processList(Object)
```

### 现在
```java
processList(Lists.newArrayList());  // 识别为: processList(List)
```

## 支持的工厂方法

### Guava Collections
- `Lists.newArrayList()` → `List`
- `Sets.newHashSet()` → `Set`
- `Maps.newHashMap()` → `Map`
- 以及其他 Guava 集合工厂方法

### Java 标准库
- `List.of()` → `List`
- `Set.of()` → `Set`
- `Map.of()` → `Map`
- `Arrays.asList()` → `List`
- `Collections.emptyList()` → `List`
- `Collections.singleton()` → `Set`
- `Collections.emptyMap()` → `Map`
- 以及其他 Collections 工具方法

### Apache Commons Collections
- `ListUtils.emptyIfNull()` → `List`
- `SetUtils.union()` → `Set`
- `MapUtils.emptyIfNull()` → `Map`

## 实现位置

- 辅助函数: `code-impact-analyzer/src/java_parser.rs::infer_collection_factory_type`
- 调用位置: `code-impact-analyzer/src/java_parser.rs::infer_argument_type_with_return_types`

## 优势

1. 更精确的方法调用签名识别
2. 更准确的影响分析结果
3. 支持主流集合库
4. 易于扩展新的工厂方法
