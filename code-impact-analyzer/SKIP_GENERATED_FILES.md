# 跳过生成文件功能

## 功能说明

解析器现在会自动跳过第一行包含 "Generated" 字样的 Java 文件。这些文件通常是由代码生成工具自动生成的，不需要进行影响分析。

## 实现细节

在 `JavaParser` 的两个主要解析方法中添加了检查逻辑：

1. `parse_file()` - LanguageParser trait 的实现
2. `parse_file_with_global_types_and_classes()` - 带全局类型信息的解析方法

### 检查逻辑

```rust
// 检查第一行是否包含 "Generated" 字样
if let Some(first_line) = content.lines().next() {
    if first_line.contains("Generated") {
        // 返回空的解析结果
        return Ok(ParsedFile {
            file_path: file_path.to_path_buf(),
            language: "java".to_string(),
            classes: vec![],
            functions: vec![],
            imports: vec![],
        });
    }
}
```

## 匹配的文件示例

以下类型的文件会被跳过：

### 1. 单行注释
```java
// Generated by some tool
package com.example;

public class GeneratedClass {
    // ...
}
```

### 2. 多行注释
```java
/* Generated code - do not edit */
package com.example;

public class AnotherGeneratedClass {
    // ...
}
```

### 3. Javadoc 注释
```java
/** Generated by annotation processor */
package com.example;

public class ProcessedClass {
    // ...
}
```

## 不会被跳过的文件

只有第一行包含 "Generated" 才会被跳过，以下情况会正常解析：

### 1. 第二行或更后面包含 Generated
```java
package com.example;
// Generated by some tool

public class NormalClass {
    // 这个文件会被正常解析
}
```

### 2. 代码中包含 Generated
```java
package com.example;

public class NormalClass {
    // This is a generated method
    public void generatedMethod() {
        // 这个文件会被正常解析
    }
}
```

## 测试

运行测试示例：

```bash
cargo run --example test_skip_generated
```

测试覆盖了以下场景：
1. 第一行包含 "Generated" 的文件 - 应该返回空结果
2. 第一行注释包含 "Generated" 的文件 - 应该返回空结果
3. 正常文件（不包含 "Generated"）- 应该正常解析
4. 第二行包含 "Generated" 的文件 - 应该正常解析

## 性能影响

- 检查只在文件开头进行，性能影响极小
- 跳过生成文件可以显著减少解析时间和内存使用
- 对于包含大量生成代码的项目，可以提升整体分析性能

## 配置

目前这是一个硬编码的行为，无需配置。如果需要自定义跳过规则，可以修改 `java_parser.rs` 中的检查逻辑。
