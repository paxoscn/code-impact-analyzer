# 方法参数完整包名实现

## 需求

Java 方法声明中的参数类型需要包含完整包名，规则与方法调用中的参数类型相同。

## 实现方案

### 核心改动

#### 1. 方法声明参数类型

修改 `extract_parameter_types` 函数，使其能够解析完整的类名（包含包名）。

**修改前：**
```rust
fn extract_parameter_types(&self, source: &str, method_node: &tree_sitter::Node) -> Vec<String>
```

**修改后：**
```rust
fn extract_parameter_types(&self, source: &str, method_node: &tree_sitter::Node, tree: &tree_sitter::Tree) -> Vec<String>
```

#### 2. 方法调用参数类型

修改以下函数以支持完整包名解析：

- `infer_argument_type_with_return_types` - 添加 `package_name` 参数
- `extract_argument_types_with_return_types` - 添加 `package_name` 参数
- `walk_node_for_calls_with_return_types` - 添加 `package_name` 参数
- `infer_method_return_type_with_map` - 添加 `package_name` 参数
- `extract_method_calls_with_return_types` - 提取并传递 `package_name`

### 类型解析规则

#### 1. 基本类型和常用类型

保持简单名称，不添加包名：
- 基本类型：`int`, `long`, `boolean`, `char`, 等
- java.lang 包：`String`, `Object`, `Integer`, 等
- 常用集合：`List`, `ArrayList`, `HashMap`, 等

#### 2. 自定义类型

按以下优先级解析：

1. **已包含包名**：如果类型名包含点号（`.`），直接使用
   ```java
   public void process(com.example.model.User user)
   // 参数类型：com.example.model.User
   
   processor.process(new com.example.model.User())
   // 调用参数类型：com.example.model.User
   ```

2. **导入映射**：从 import 语句中查找
   ```java
   import com.example.model.User;
   public void process(User user)
   // 参数类型：com.example.model.User
   
   User user = new User();
   processor.process(user)
   // 调用参数类型：com.example.model.User
   ```

3. **同包类**：假设在同一个包中
   ```java
   package com.example.service;
   public void process(UserService service)
   // 参数类型：com.example.service.UserService
   
   UserService service = new UserService();
   processor.process(service)
   // 调用参数类型：com.example.service.UserService
   ```

4. **无法解析**：返回简单类名
   ```java
   public void process(UnknownType obj)
   // 参数类型：UnknownType
   ```

## 测试示例

### 方法声明测试

```java
package com.example.service;

import com.example.model.User;
import com.example.model.Address;
import java.util.List;

public class UserService {
    // 导入的类型
    public void processUser(User user, Address address) { }
    
    // 常用类型
    public void processUsers(List users, String name) { }
    
    // 完整包名类型
    public void processData(com.example.model.User user, java.util.List list) { }
    
    // 基本类型
    public void processNumbers(int count, String text, boolean flag) { }
}
```

**解析结果：**
```
✓ processUser(com.example.model.User,com.example.model.Address)
✓ processUsers(List,String)
✓ processData(com.example.model.User,java.util.List)
✓ processNumbers(int,String,boolean)
```

### 方法调用测试

```java
package com.example.service;

import com.example.model.User;
import com.example.model.Address;

public class UserService {
    public void testCalls() {
        // 变量参数
        User user = new User();
        processUser(user);
        
        // 对象创建参数
        processUser(new User());
        
        // 类型转换参数
        Object obj = new Object();
        processUser((User) obj);
        
        // 同包类型
        UserData data = new UserData();
        processData(data);
        
        // 完整包名类型
        processAddress(new com.example.model.Address());
    }
}
```

**解析结果：**
```
✓ processUser(com.example.model.User) - 变量参数
✓ processUser(com.example.model.User) - 对象创建参数
✓ processUser(com.example.model.User) - 类型转换参数
✓ processData(com.example.service.UserData) - 同包类型
✓ processAddress(com.example.model.Address) - 完整包名类型
```

**方法声明：**
- `type_identifier` - 简单类型
- `generic_type` - 泛型类型
- `array_type` - 数组类型
- `scoped_type_identifier` - 完整包名类型
- `integral_type`, `floating_point_type`, `boolean_type` - 基本类型

**方法调用：**

- `object_creation_expression` - 对象创建

  - `type_identifier` - 简单类型
  - `generic_type` - 泛型类型
- `method_invocation` - 方法调用（返回类型）

## 实现细节

### 避免重复包名

**问题：** 对于已经包含完整包名的类型（如 `com.example.model.User`），不应该再次添加包名前缀。

**解决方案：** 在 `resolve_full_class_name` 中添加检查：

```rust
// 如果已经包含点号，说明已经是完整类名
if simple_name.contains('.') {
    return simple_name.to_string();
}
```

### 类型解析流程

```
参数类型
    ↓
是基本类型或常用类型？
    ↓ 是
    返回简单名称
    ↓ 否
已包含包名（有点号）？
    ↓ 是
    返回原类型
    ↓ 否
在导入映射中？
    ↓ 是
    返回完整类名
    ↓ 否
在同一个包中？
    ↓ 是
    返回 包名.类名
    ↓ 否
    返回简单类名
```
