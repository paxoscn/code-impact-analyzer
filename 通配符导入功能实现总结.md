# 通配符导入支持实现总结

## 需求

对于没有明确import的非基础java类，如`Foo foo;`，若代码中有`import bar.*;`，则先在全局索引中查找`bar.Foo`类，若找到了就用作Foo的全限定名，若找不到再使用当前代码的包名。

## 实现内容

### 1. 核心功能

#### 全局类索引集成
- 在第一遍索引时，构建全局类索引（`global_class_index`）
- 在第二遍索引时，将全局类索引传递给解析器
- 使用全局类索引精确解析通配符导入的类

#### 解析优先级（使用全局索引）
1. **明确的import语句**：如果有`import com.example.Foo;`，直接使用`com.example.Foo`
2. **通配符导入包（全局索引验证）**：如果有`import bar.*;`，在全局索引中查找`bar.Foo`，找到则使用
3. **当前包**：如果以上都找不到，使用当前代码的包名

### 2. 新增/修改的方法

#### `code_index.rs`
- 修改`index_workspace_two_pass`：构建并使用全局类索引
- 修改`index_workspace`：构建并使用全局类索引

#### `java_parser.rs`

**核心解析方法：**
- `resolve_full_class_name_with_wildcards`：使用全局索引精确解析类名
- `resolve_full_class_name_with_wildcard_fallback`：启发式方法（向后兼容）

**字段类型提取：**
- `extract_field_types_with_global_index`：使用全局索引提取字段类型
- `extract_method_parameter_types_with_wildcards_and_index`：使用全局索引提取方法参数类型
- `extract_local_variable_types_with_wildcards_and_index`：使用全局索引提取本地变量类型

**方法调用提取：**
- `extract_method_calls_with_return_types_and_index`：使用全局索引提取方法调用

**API方法：**
- `parse_file_with_global_types_and_classes`：接受全局类索引的解析方法
- `parse_file_with_global_types`：保持向后兼容，内部调用新方法

### 3. 实现策略

#### 精确方法（当前实现）

使用全局类索引进行精确解析：

1. 在第一遍索引时，收集所有类的完整限定名
2. 在第二遍索引时，使用全局类索引验证通配符导入
3. 只有在全局索引中找到类时，才使用通配符导入的包

**优点：**
- 精确解析，避免歧义
- 正确处理多个通配符导入的情况
- 可以区分同名类在不同包中的情况

**实现：**
```rust
// 第一遍：构建全局类索引
let mut global_class_index = rustc_hash::FxHashMap::default();
for parsed_file in &parsed_files_pass1 {
    for class in &parsed_file.classes {
        global_class_index.insert(class.name.clone(), class.name.clone());
    }
}

// 第二遍：使用全局类索引解析
match java_parser_concrete.parse_file_with_global_types_and_classes(
    &content, 
    file_path, 
    &global_return_types, 
    &global_class_index
) {
    Ok(parsed) => Some(parsed),
    Err(e) => None,
}
```

#### 启发式方法（向后兼容）

不使用全局索引，使用启发式方法：

- 如果有通配符导入，优先使用第一个通配符导入的包
- 适用于没有全局索引的场景

**保留原因：**
- 向后兼容
- 单文件解析时不需要全局索引
- 在大多数情况下也是正确的

## 使用示例

### 两遍索引（自动使用全局索引）

```rust
let mut code_index = CodeIndex::new();
let parsers = vec![Box::new(JavaParser::new()?) as Box<dyn LanguageParser>];

// 两遍索引会自动构建和使用全局类索引
code_index.index_workspace_two_pass(workspace_path, &parsers)?;
```

### 单文件解析（使用全局索引）

```rust
let parser = JavaParser::new()?;

// 构建全局类索引
let mut global_class_index = rustc_hash::FxHashMap::default();
global_class_index.insert("bar.Foo".to_string(), "bar.Foo".to_string());

// 使用全局索引解析
let result = parser.parse_file_with_global_types_and_classes(
    content, 
    file_path, 
    &global_return_types,
    &global_class_index
)?;
```

## 性能考虑

1. **第一遍索引**：额外收集类名，开销很小
2. **第二遍索引**：查找全局索引，O(1)时间复杂度
3. **内存开销**：全局类索引大小约为类数量 × 平均类名长度
