# 跨文件类型推断说明

## 问题回答

**问题：解析调用的参数类型时用到索引中其他类的 return_type 了吗？**

**回答：是的，现在已经完全实现了！使用两遍索引策略。**

## 最新实现（两遍索引策略）

### 实现方式

我们实现了两遍索引策略来支持跨文件类型推断：

**第一遍：提取方法签名和返回类型**
- 快速解析所有文件
- 提取每个方法的签名和返回类型
- 构建全局返回类型映射

**第二遍：使用全局类型信息重新解析**
- 使用全局返回类型映射重新解析所有文件
- 推断跨文件的方法调用参数类型
- 生成更精确的方法调用信息

### 使用方法

```rust
use code_impact_analyzer::{CodeIndex, JavaParser};
use code_impact_analyzer::language_parser::LanguageParser;

let java_parser = Box::new(JavaParser::new().unwrap());
let parsers: Vec<Box<dyn LanguageParser>> = vec![java_parser];

let mut index = CodeIndex::new();

// 使用两遍索引（支持跨文件类型推断）
index.index_workspace_two_pass(workspace_path, &parsers)?;

// 或者使用单遍索引（更快，但跨文件类型不精确）
// index.index_workspace(workspace_path, &parsers)?;
```

## 实际示例

### 场景：跨文件方法调用

```java
// 文件 A: UserRepository.java
package com.example;

public class UserRepository {
    public User findById(String id) {
        return new User();
    }
    
    public Address getAddress(User user) {
        return user.getAddress();
    }
}

// 文件 B: UserService.java
package com.example;

public class UserService {
    private UserRepository repository;
    
    public void processUser(String userId) {
        // 跨文件调用：repository.findById() 返回 User
        // 两遍索引后，processor.process() 的参数类型被正确推断为 User
        processor.process(repository.findById(userId));
    }
    
    public void processAddress(String userId) {
        // 嵌套跨文件调用：
        // repository.findById() 返回 User
        // repository.getAddress(User) 返回 Address
        // formatter.format() 的参数类型被正确推断为 Address
        formatter.format(repository.getAddress(repository.findById(userId)));
    }
}
```

### 索引结果对比

**单遍索引：**
- `processor.process(Object)` - 参数类型不精确
- `formatter.format(Object)` - 参数类型不精确

**两遍索引：**
- `processor.process(User)` - 参数类型精确 ✓
- `formatter.format(Address)` - 参数类型精确 ✓

## 性能考虑

### 单遍索引
- **速度**：快，只解析一次
- **精度**：文件内类型推断精确，跨文件类型为 Object
- **适用**：大型项目（> 10万行），性能优先

### 两遍索引
- **速度**：约为单遍的 2 倍
- **精度**：文件内和跨文件类型推断都精确
- **适用**：中小型项目（< 10万行），精度优先

## 工作原理

```

│ 第一遍：提取方法签名和返回类型                          │

│   - findById(String) -> User                            │
│   - getAddress(User) -> Address                         │
│                                                         │
│ UserService.java                                        │
│   - processUser(String) -> void                         │
│   - processAddress(String) -> void                      │
└─────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────┐
│ 构建全局返回类型映射                                    │
├─────────────────────────────────────────────────────────┤
│ com.example.UserRepository::findById(String) -> User    │
│ com.example.UserRepository::getAddress(User) -> Address │
│ com.example.UserService::processUser(String) -> void    │
│ com.example.UserService::processAddress(String) -> void │
└─────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────┐
│ 第二遍：使用全局类型信息重新解析                        │
├─────────────────────────────────────────────────────────┤
│ UserService.java                                        │
│   processUser:                                          │
│     - repository.findById(userId)                       │
│       查询全局映射 -> 返回 User                         │
│     - processor.process(User)  ← 参数类型精确！         │
│                                                         │
│   processAddress:                                       │
│     - repository.findById(userId)                       │
│       查询全局映射 -> 返回 User                         │
│     - repository.getAddress(User)                       │
│       查询全局映射 -> 返回 Address                      │
│     - formatter.format(Address)  ← 参数类型精确！       │
└─────────────────────────────────────────────────────────┘
```
