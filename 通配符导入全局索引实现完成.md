# 通配符导入全局索引实现完成

## 实现概述

已成功实现使用全局索引精确解析通配符导入的功能。对于没有明确import的非基础Java类（如`Foo foo;`），当代码中有`import bar.*;`时，系统会：

1. 在全局索引中查找`bar.Foo`类
2. 如果找到，使用`bar.Foo`作为完整限定名
3. 如果找不到，回退到当前代码的包名

## 核心改动

### 1. code_index.rs

在两遍索引中集成全局类索引：

```rust
// 第一遍：构建全局类索引
let mut global_class_index = rustc_hash::FxHashMap::default();
for parsed_file in &parsed_files_pass1 {
    for class in &parsed_file.classes {
        global_class_index.insert(class.name.clone(), class.name.clone());
    }
}

// 第二遍：使用全局类索引解析
match java_parser_concrete.parse_file_with_global_types_and_classes(
    &content, 
    file_path, 
    &global_return_types, 
    &global_class_index
) {
    Ok(parsed) => Some(parsed),
    Err(e) => None,
}
```

### 2. java_parser.rs

新增/修改的关键方法：

- `parse_file_with_global_types_and_classes`: 接受全局类索引
- `extract_method_calls_with_return_types_and_index`: 使用全局索引提取方法调用
- `extract_field_types_with_global_index`: 使用全局索引提取字段类型
- `extract_method_parameter_types_with_wildcards_and_index`: 使用全局索引提取方法参数
- `extract_local_variable_types_with_wildcards_and_index`: 使用全局索引提取本地变量
- `resolve_full_class_name_with_wildcards`: 使用全局索引精确解析类名

## 解析流程

### 第一遍索引
1. 解析所有Java文件
2. 提取所有类的完整限定名
3. 构建全局类索引：`HashMap<String, String>`
4. 构建全局方法返回类型映射

### 第二遍索引
1. 对每个Java文件重新解析
2. 传入全局类索引和方法返回类型映射
3. 在解析类型时：
   - 提取通配符导入列表
   - 对于未明确导入的类，在通配符导入的包中查找
   - 使用全局索引验证类是否存在
   - 如果存在，使用该包名；否则回退到当前包

## 示例

### 输入代码
```java
package com.example.service;

import com.example.model.*;
import com.example.dto.*;

public class UserService {
    public void process() {
        User user = new User();           // 在 com.example.model 中找到
        UserDTO dto = new UserDTO();      // 在 com.example.dto 中找到
        Helper helper = new Helper();     // 都找不到，使用当前包
    }
}
```

### 全局索引
```
com.example.model.User -> com.example.model.User
com.example.dto.UserDTO -> com.example.dto.UserDTO
com.example.service.Helper -> com.example.service.Helper
```

### 解析结果
- `User` -> `com.example.model.User` ✅ (在第一个通配符导入中找到)
- `UserDTO` -> `com.example.dto.UserDTO` ✅ (在第二个通配符导入中找到)
- `Helper` -> `com.example.service.Helper` ✅ (通配符导入中都找不到，回退到当前包)

## 性能影响

1. **第一遍索引**：额外收集类名，开销很小（每个类一次插入操作）
2. **第二遍索引**：查找全局索引，O(1)时间复杂度
3. **内存开销**：全局类索引大小 ≈ 类数量 × 平均类名长度（通常几MB）

## 使用方式

### 自动使用（推荐）

使用两遍索引时，自动使用全局索引：

```bash
cargo run -- index --workspace /path/to/project
```

### 手动使用

```rust
let parser = JavaParser::new()?;
let mut global_class_index = rustc_hash::FxHashMap::default();

// 构建全局索引
for class in all_classes {
    global_class_index.insert(class.name.clone(), class.name.clone());
}

// 使用全局索引解析
let result = parser.parse_file_with_global_types_and_classes(
    content, 
    file_path, 
    &global_return_types,
    &global_class_index
)?;
```
